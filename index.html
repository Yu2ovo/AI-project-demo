PROJECT OVERVIEW

        
项目展示重点

        不直接暴露 API Key，不全量部署后端；通过轻量页面展示核心业务链路和技术方案。
      

      

        

          
统一模型调用

          
基于 Spring AI 封装 ChatClient 调用入口，方便接入通义千问等模型，并支持后续切换模型服务。

        

        

          
RAG 知识库增强

          
针对模型缺少业务私有知识的问题，先召回相关知识片段，再注入上下文生成回答，降低凭空回答风险。

        

        

          
多轮对话记忆

          
通过会话 ID 维护上下文，使客服能够理解用户连续追问，而不是每轮都从零开始回答。

        

        

          
Tool Calling / MCP

          
将文件操作、网页抓取、联网搜索、图片检索等能力封装为工具，使 AI 能够调用外部能力完成复杂任务。
